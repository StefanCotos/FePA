1. Introduction

1.1 Purpose

We aim to launch a web application to monitor the behavior of stray or homeless animals, animals that pose a danger to humans and beyond. With this application it is easier to monitor some animals.

1.2 Site Conventions

In creating the website, we used:

Font: Trebuchet MS, sans-serif

Colors: lightslategray, red, green, #9f917e, #fde8cd

Libraries: firebase/php-jwt, cloudinary/cloudinary_php, vlucas/phpdotenv, phpmailer/phpmailer, laminas/laminas-feed , jsPDF to generate PDF, google charts to make charts, Open Street Map to display a map and others...

We used the REST API architecture to make requests

1.3 Intended Audience

Primarily young adults and adults, followed by a smaller percentage of children and the elderly.

1.4 Product Scope

Our aim is to provide assistance to people through this application, as animals can pose a significant danger if they are not properly treated or cared for. Before taking a stroll or heading to a particular area, people can check our application to see if there have been any recent reports in those areas.

Our high-level goals are to:

Making the website accessible worldwide. Reducing animal attacks on humans. Creating a space where animals and humans can coexist without disturbing each other.

2. Overall Description

2.1 Product Perspective

This website is an original one, and the idea behind it makes it very useful in the everyday lives of those who will use the site for personal or public purposes.

2.2 Product Functions

- viewing reports

- creating reports

- environmental study using

- follow reports like RSS feeds

2.3 User Classes and Characteristics

Young people: They use their phones daily and will want to use this page because they can easily find information and provide information to others just as easily.

Adults: Being responsible, they will want to help others, so they will make reports to assist others.

2.4 Operating Environment

Because it is a website, it will run on any device that can access a browser, regardless of the type of browser or its version. You can access it on any device that has internet access.

2.5 Design and Implementation Constraints

One of the issues that may be encountered is that we rely on people worldwide to create reports, but they may create false or inaccurate reports, which can be challenges during development.

Another problem would be the need for strong security to ensure that no one can access data about other users.

2.6 User Documentation

The website includes a help page that explains everything you need to know about using this website. This information can help you interact better with the site.

3. Structure

3.1 The header and footer

The two components are present on all pages, with the header containing the menu with all accessible pages and the logo that takes you to the homepage. In the footer, you'll find the logo and some contact details alongside another menu containing all the existing pages on the site.

3.2 The admin page

The admin page can currently only be accessed from the homepage, but in the future, it will only be accessible after logging in, only by people with admin power. This way, a person can prove that they are an admin and will have access to the page and options.
This page contains a table with all existing users, all existing postings and a table with all the requests for postings, preventing erroneous information from being posted through reports.

3.3 The about us page

The 'About' page contains brief information about the application, its role, as well as some details about the founders of the website.

3.4 The report page

This page contains two components:
1.The form that needs to be filled out entirely in order to create the post
2.The map that helps you better visualize the places where other animals have been reported

To create a report post, you need to follow these steps:
1. You must have an account on this website.
2. You need to fill in all fields, as they are all mandatory.
After completing these requirements, the post should be successfully submitted.

3.5 The see reports page

On the 'see reports' page, you will find all the animals for which reporting requests have been completed. This list operates with a right scroll, allowing you to navigate through multiple reports.
If you click on an animal post, you will be able to see the reporting description to view the exact details about that animal.

3.6 The statistics page

On the statistics page, we have 3 important statistics:
1. Areas - the areas where the most reports have been made
2. Dangerous - the danger posed by the most animals
3. Types - the most reported type of animal

All of these help us and you to learn more about the areas with the most reported animals and how dangerous
they are.

3.7 The contact page

On the contact page, there is a form where if you fill it out with the requested information, you will be able to email us regarding an issue you encountered on this website and couldn't resolve.
Also on this page, you will find the location of our headquarters, so you can even come to discuss certain serious issues related to this application.

3.8 The log-in page

On the login page, you can log into your account or go to the sign-up page to create an account by clicking on the text 'Sign up' below the login button.
If you have forgotten your password, you will be redirected to a page where you will need to enter the email associated with your account, and you will receive a password reset code via email.

3.9 The sign-up page

On the sign-up page, you create an account. To do this, you need to fill out the form with your personal details as requested. After completing them, ensure that all information is correct and that you have filled in all required fields.
If you remember that you already have an account, you can quickly return from the sign-up page to the login page by clicking on the text 'Click here' below the Sign-up button.

4. System Features

4.1 System Feature

You can create reports that are saved in the database and are displayed in several formats (normally site-specific and RSS news feed). You can see the reports, and information about them, as well as who made the announcement.

4.1.1 Description and Priority

Creating a well-secured and highly structured database to streamline post creation and data processing.

Priority 9

4.1.2 Stimulus/Response Sequences

The user will fill out the form, and the website will generate that post on the 'See Reports' page.

4.1.3 Functional Requirements

The user will need to use a browser with the website open and be logged into their account. Then, the user will fill out the form and receive simple error messages if they haven't entered required information in the form.

5. Other Nonfunctional Requirements

5.1 Performance Requirements

This website does not require special performance from users, as long as they can run a browser on their devices, they will be able to access the FePA website.

5.2 Safety Requirements

In the photos published on the website, the faces of individuals who were passing by at that moment may be visible. For the safety of those individuals, their faces will be blurred before the reports are made public to avoid causing any issues.

5.3 Security Requirements

To prevent information leakage from our database, it needs to be carefully encrypted. To prevent the publication of reports with incorrect content, site admins are responsible for verifying the reports.

5.4 Software Quality Attributes

The quality attributes of the software are:

Accessibility: The site is accessible from anywhere.

Convenience: You can create a post with just a few clicks.

Quality: Posts will go through an admin filter to determine if they are accepted or not.

Speed of information transmission: You can find the latest animal reports in your area with just a few scrolls.

5.5 Business Rules

Users can only create reports and inspect any information made public on the site. Admins can do everything a regular user can, but they can also inspect data about any user and any future post.
